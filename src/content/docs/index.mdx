<Aside>
Remember, Ghostty is a passion project led by one person and maintained by volunteers.
</Aside>

Non-features:

- no sixel support ([#2496](https://github.com/ghostty-org/ghostty/discussions/2496))
- macOS 12 is not supported ([#3696](https://github.com/ghostty-org/ghostty/discussions/3696#discussioncomment-11703049))

The roadmap includes:

- scrollbars ([#111](https://github.com/ghostty-org/ghostty/issues/111))
- Windows support ([#2563](https://github.com/ghostty-org/ghostty/discussions/2563))

{/* - publishing `libghostty`, a source-of-truth implementation of terminal specs ([ghostty.org/docs/about](https://ghostty.org/docs/about#libghostty)) */}

Some confusing things:

- command-line actions (e.g. `+list-themes`) are completely different than keybind actions (e.g. `reload_config`).
Pretend the command-line actions are just called subcommands ([#5622](https://github.com/ghostty-org/ghostty/discussions/5622#discussioncomment-12095784))
- keybind actions (e.g. `reload_config`) cannot be activated from the command line. 
Those actions are only accessible through keyboard inputs ([#2353](https://github.com/ghostty-org/ghostty/discussions/2353))
{/* - most of these lesser-known items are primarily in discord messages, with several repeated in github discussions, and a few making their way to the official documentation */}
